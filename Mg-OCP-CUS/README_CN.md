- # 镁合金 CUS 缺陷表面数据处理操作指南

  本项目致力于从零构建一个包含 **15,421** 个镁合金配位不饱和点（CUS）缺陷表面的高保真数据集 (Mg-OCP-CUS)

  ## 📂 目录结构与状态说明

  ```text
  .
  ├── dataset_pt/           # 【终点】存放最终生成的 PyTorch .pt 张量文件
  ├── seed_structures/      # 【数据中转站】存储流水线中各个环节的晶体结构文件
  │   ├── lhs_shards/       # 【待生成】预留给 Step 3 的 125 个缺陷分片文件
  │   └── relaxed_shards/   # 【待生成】预留给 Step 4 的弛豫后终态结构及日志
  ├── vesta_view/           # 【可视化工具】存放用于在 VESTA 软件中查看晶体构型的 .cif 文件
  ├── oqmd_cifs/            # 【前置原始数据】存放从 OQMD 数据库拉取的亚稳态固溶体 .cif 文件 (53 个)
  ├── icsd_cifs/            # 【前置原始数据】存放从 ICSD/COD 数据库手动下载的实验级高精度 .cif 文件 (22 个)
  ├── fetch_seeds.py        # 第一步：多源基础结构池融合获取 (MP + OQMD + ICSD)
  ├── cleave_slabs.py       # 第二步：纯净晶面切分与真空层构建
  ├── lhs_defects_sharded.py # 第三步：LHS 抽样缺陷演化与分片生成
  ├── cusrep_relax.py       # 第四步：调用 CUS-Rep 执行物理弛豫与能量标签计算
  ├── feature_extraction.py # 第五步：纯矩阵驱动的物理特征提取与 .pt 张量化
  └── helper_oqmd.py        # 【辅助脚本】OQMD 亚稳态固溶体自动化爬虫引擎
  ```

  ------

  ## 🚀 全链路执行手册 (Step-by-Step)

  请严格按照以下顺序在终端中执行脚本。**前置条件**：请确保你的 `oqmd_cifs/` (可通过运行 `helper_oqmd.py` 自动抓取) 和 `icsd_cifs/` (需手动下载) 文件夹中已经准备好了所需的前置 `.cif` 晶体文件。

  ### 🟢 阶段 1：基础数据生成 (Steps 1-3)

  **第一步：种子结构融合**

  - **命令**: `python fetch_seeds.py`
  - **目标**: 提取并融合 Materials Project 的基态数据与本地的亚稳态、实验级数据。
  - **产出**: `seed_structures/all_mixed_seeds.json` 和对应的日志文件。

  **第二步：纯净表面切分**

  - **命令**: `python cleave_slabs.py`
  - **目标**: 沿不同密排面切分种子结构，并添加真空层，生成理想纯净平板。
  - **产出**: `seed_structures/mg_cleaved_slabs.json`。

  **第三步：LHS 缺陷抽样与并行演化**

  - **命令**: `python lhs_defects_sharded.py`
  - **目标**: 引入空位、取代、吸附等复杂缺陷，并行生成目标规模的初始构型。
  - **产出**: 在 `seed_structures/lhs_shards/` 目录下生成 **125 个** 分片文件（总计约 125,000 个结构）。

  ------

  ### 🟡 阶段 2：神经力场弛豫 (Step 4)

  **第四步：调用 CUS-Rep 黑盒优化器进行弛豫** 在跑全量数据前，请确保代码内没有开启截断测试（即**不要**限制 `unrelaxed_slabs = unrelaxed_slabs[:2]`）。

  - **命令**: `python cusrep_relax.py`
  - **目标**: 逐个读取 125 个分片，利用 CUS-Rep 代理模型结合 ASE 优化器寻找局部能量最低点，并计算广义表面缺陷形成能。
  - **耗时预警**: 处理 125,000 个结构的弛豫极其耗时，预计需要 **6-7 天**（视具体硬件算力而定），建议在服务器上使用 `nohup` 或 `tmux` 后台挂机运行。
  - **产出**: 在 `seed_structures/relaxed_shards/` 目录下生成对应的 125 个弛豫后分片文件及 JSON 日志。

  ------

  ### 🔵 阶段 3：张量化与数据集闭环 (Step 5)

  **第五步：物理特征提取** 当 Step 4 彻底跑完（或跑完部分分片想要先提取时）：

  - **命令**: `python feature_extraction.py`
  - **目标**: 基于纯 Numpy 的 MIC 周期性矩阵算法，提取 Voronoi 拓扑图、广义配位数（GCN）和坐标场，并将其转化为 PyTorch 可读的 `.pt` 张量字典。
  - **产出**: 在 `dataset_pt/` 目录下生成从 `mg_cus_0.pt` 到 `mg_cus_124999.pt` 的张量文件集。

  ------

  ## ⚠️ 核心注意事项与排错

  1. **断点续跑机制**: `cusrep_relax.py` 内置了断点续跑功能。如果 Step 4 因为断电或显存溢出中途停止，直接重新运行该脚本即可，它会自动跳过已经成功生成的 `mg_relaxed_part_XX.json` 文件。
  2. **锁死数据一致性**: 一旦完成了 Step 3 并生成了 LHS 分片，**绝对不要**再回去修改 Step 3 的参数并重新生成。否则会导致之前生成的结构与后续的特征提取发生严重的错位失真。
  3. **极高的存储要求**: 125,000 个 `.pt` 张量文件由于包含高维图结构和掩码，预计需要占用 **40-60 GB** 的硬盘空间，请提前清理好足够的磁盘容量。
  4. **终极数据漏斗筛选**: 经过 CUS-Rep 的收敛性检查、异常构型剔除（如弛豫过程中原子过度重叠导致解体）以及最终的物理一致性清洗，我们最终将从这批海量样本中，精准筛选出论文所需的 **15,421** 个高质量 CUS 缺陷表面结构。

  由于文件大小限制和后续研究，仓库中提供的是用于保障代码完全可复现的代表性子集。15421个完整数据可以通过仓库里的代码全自动生成。
